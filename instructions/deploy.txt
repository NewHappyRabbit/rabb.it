This instructions file shows the process to deploy the app to production for the first time.

For local installs without domain (on customer PC, not VPS):
1. Install nginx and config using the instructions in instructions\nginx.txt
2. Install mkcert and create SSL certificates following the instructions in instructions\local_ssl.txt
3. Make nginx autostart on boot (via the instructions)
4. Make node server autostart on boot:
- Windows: https://npmjs.org/package/qckwinsvc
- Linux: https://www.npmjs.com/package/pm2

---------
For VPS installs with domain on Debian:
1. Install nginx: https://www.digitalocean.com/community/tutorials/initial-server-setup-with-debian-11

2. Config nginx: https://www.digitalocean.com/community/tutorials/how-to-install-nginx-on-debian-11
Use the server blocks from instructions\nginx_server.txt

2. Install certbot for SSL: https://certbot.eff.org/instructions?ws=nginx&os=pip
- Do "sudo apt install python3-certbot-nginx" after Step 2.

3. Generate SSH keys on VPS for:
3.1 Access from VPS to Github (for pull requests): https://docs.github.com/en/authentication/connecting-to-github-with-ssh/generating-a-new-ssh-key-and-adding-it-to-the-ssh-agent
- Run the following commands:
-- Generate SSH key: ssh-keygen -t ed25519 -C "your@email.com"
-- Open SSH agent: eval "$(ssh-agent -s)"
-- Add SSH key to agent: ssh-add ~/.ssh/id_ed25519
3.2 Access from Github actions to VPS:
- Generate SSH key: ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
- Copy the contents of id_rsa.pub to .ssh/authorized_keys
- Add the contents of id_rsa to Repository > Settings > Secrets > Actions > Add a new secret > SSH_PRIVATE_KEY

4. Add the SSH key to Github:
- Copy the contents of id_ed25519.pub to https://github.com/settings/keys
- Copy the contents of id_ed25519 to Repository > Settings > Secrets > Actions > Add a new secret > SSH_PRIVATE_KEY
- Create a personal access token in github
-- Open https://github.com/settings/tokens
-- Generate new access token (classic)
--- Expiration: never
--- Select "repo"
--- Copy the key (ghp_....)

4. Install git: sudo apt install git-all

5. Git clone the repository
- Run: git clone -b branch --single-branch https://username:token@github.com/username/repo.git
-- Replace username, token with your personal access token, repo with the name of the repository, and branch with the name of the branch
-- Example: git clone -b live --single-branch https://NewHappyRabbit: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@github.com/NewHappyRabbit/rabb.it.git
--- "-b branch" selects which branch to clone
--- "--single-branch" clones only the specified branch

6. Cd to the rabb.it folder and run:
- git remote set-url origin git@github.com:username/repo.git
- Example: git remote set-url origin git@github.com:NewHappyRabbit/rabb.it.git

7. Install node: https://github.com/nodesource/distributions?tab=readme-ov-file#using-debian-as-root-nodejs-20

8. Install dependencies: npm i

9. Setup node to autostart with PM2: https://www.npmjs.com/package/pm2
- Install globally: npm install pm2 -g
- Start the rabb.it/entry.cjs file and name the process as "node": pm2 start entry.cjs --name node -i max --log pm2.log
- Set the process to startup with machine: pm2 startup
- Save: pm2 save

10. Run the frontend once: npm run web-prod
